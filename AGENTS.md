# 디자인 레퍼런스 아카이브 작업지시서

## 목적

이 폴더는 웹·앱 콘텐츠 디자인, 브랜드 디자인, AI 디자인, 비주얼 그래픽 및 기타 디자인 영감을 수집하고 다시 찾기 쉽게 정리하는 개인 아카이브다. 자료는 15년차 UI 디자이너의 관점에서, 완성도·새로움·실무 적용 가능성이 높은 것을 우선한다.

## 최종적으로 보여 주는 방식

사용자가 보는 결과물은 표나 긴 텍스트가 아니라 **이미지 미리보기와 원본 링크가 중심인 레퍼런스 카드형 아카이브**여야 한다.

각 레퍼런스 카드에는 아래 정보를 포함한다.

- 대표 이미지 또는 스크린샷(가능할 때만)
- 제목 또는 짧은 설명
- 원본 링크
- 분야: `웹·앱 UI`, `브랜드`, `AI 디자인`, `비주얼 그래픽`, `기타`
- 스타일/핵심 특징 1~2개
- 검색용 태그
- 수집일 (`YYYY-MM-DD`)
- 왜 참고할 만한지에 대한 짧은 디자이너 메모

이미지를 안전하게 저장할 수 없거나 저작권·접근 제약이 있는 경우에는 이미지를 무단 복제하지 말고, 링크와 썸네일/이미지 확인 위치를 제공한다.

## 수집 기준

- 매일 수집 작업을 요청받으면 최신성과 디자인 품질을 함께 고려해 큐레이션한다.
- 단순히 유행하는 결과물보다 정보 구조, 사용자 흐름, 타이포그래피, 브랜드 일관성, 접근성, 인터랙션, AI 활용 방식 등 실무적으로 배울 점이 있는 자료를 우선한다.
- 한 번에 너무 많은 자료를 쌓지 않는다. 중복을 제거한 뒤 분야가 고르게 섞인 소수의 강한 레퍼런스를 제안한다.
- 출처와 원본 링크를 가능한 한 반드시 남긴다. 출처를 확인할 수 없으면 신뢰도 낮음으로 표시하거나 제외한다.

## 분류와 태그

- 자료는 먼저 `분야별`, 다음으로 `스타일별`로 탐색할 수 있게 정리한다.
- 모든 항목에 수집일을 남긴다.
- 태그는 짧고 일관된 한국어 중심 키워드를 사용한다. 예: `#온보딩`, `#대시보드`, `#미니멀`, `#3D`, `#에디토리얼`, `#생성형AI`.
- 기존 항목의 링크, 프로젝트, 시각적 결과물, 핵심 콘셉트가 실질적으로 같으면 새로 추가하지 않는다.

## 안전 및 파일 관리

- 외부 서비스에 파일이나 아카이브 내용을 업로드·공유·게시하지 않는다.
- 원본 파일을 임의로 삭제하거나 덮어쓰지 않는다.
- 파일명과 폴더명은 한눈에 이해할 수 있게 짓고, 날짜는 `YYYY-MM-DD` 형식을 사용한다.
- 저작권이 있는 이미지·파일은 필요한 범위에서만 다루고 원 출처를 보존한다.

## 작업 완료 기준

매 수집 작업이 끝나면 다음을 확인한다.

1. 새 자료가 카드형 화면/문서에서 이미지(가능한 경우)와 링크로 바로 확인되는가.
2. 분야, 스타일, 태그, 수집일, 디자이너 메모가 채워졌는가.
3. 기존 아카이브와 중복되지 않는가.
4. 전체 목록 또는 인덱스에서 새 자료를 찾을 수 있는가.
5. 사용자에게 추가한 자료 수, 주요 분야, 제외한 중복 여부를 2~4문장으로 요약했는가.

## 커뮤니케이션

- 설명과 결과 보고는 한국어로, 짧고 명확하게 작성한다.
- 확신하기 어려운 출처·저작권·이미지 사용 가능 여부는 추측하지 말고 표시한다.
- 자동으로 외부 업로드, 삭제, 공개하지 않는다.
